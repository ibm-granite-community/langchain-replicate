# AGENTS.md

This file provides guidance to agents when working with code in this repository.

## Testing

- Tests are integration-style and require the `REPLICATE_API_TOKEN` environment variable to be set to a valid API token. If there is a `.env` file present, use the `--env-file=.env` flag when using `uv run`.
- Run all tests with `uv run pytest tests`.
- Run one file with `uv run pytest tests/test_chat_models.py`.
- Run one test with `uv run pytest tests/test_chat_models.py::TestChat::test_invoke`.
- Slow embedding coverage is opt-in via `uv run pytest -m slow tests/test_embeddings.py`.
- Tests use `assertpy` for assertions (not `assert` statements).

## Lint / Type Check

- Lint: `uv run ruff check` (auto-fix: `uv run ruff check --fix`)
- Format: `uv run ruff format`
- Type check: `uv run pyrefly check`
- Inline pyrefly suppressions use `# pyrefly: ignore [missing-attribute]` syntax, not `# type: ignore`.

## Version Control

- Always create signed tags using `git tag -s` when tagging releases; unsigned tags are not permitted.

## Code Style

- All source files begin with `# SPDX-License-Identifier: MIT`.
- All source files use `from __future__ import annotations` as the first import.
- Max line length is 180 (non-standard); do not wrap lines that fit within 180 chars.
- Active ruff rule sets: B, E, W, F, Q, I (isort), UP (pyupgrade), SIM, TID, N (pep8-naming).

## Project-specific patterns

- Always pass prediction payloads through [`_adjust_prediction_input()`](src/langchain_replicate/_base.py:52); it rewrites unsupported `tool_choice="any"` to `"required"` and JSON-serializes dict or named tool choices before hitting Replicate.
- Use [`_json_dumps`](src/langchain_replicate/_base.py:43) instead of [`json.dumps`](json/__init__.py) when serializing tool/function payloads; separators and NaN handling are intentionally aligned with vLLM-style JSON responses.
- Normalize tool-call argument strings with [`_normalize_tool_arguments()`](src/langchain_replicate/chat_models.py:79); chat models expect malformed LLM outputs such as Python dict strings, nested JSON strings, and repaired JSON.
- Do not hardcode prompt/input field names for models. [`Replicate._create_prediction_input()`](src/langchain_replicate/llms.py:101) and [`ReplicateEmbeddings._create_prediction_input()`](src/langchain_replicate/embeddings.py:58) derive the first input key from the model OpenAPI schema.
- Deployment identifiers (`owner/name:deployment`) mutate [`ReplicateBase.model`](src/langchain_replicate/_base.py:78) to the released model name when [`ReplicateBase._deployment`](src/langchain_replicate/_base.py:111) is resolved; avoid assuming the original model string remains unchanged.
- Stop-sequence handling is schema-dependent: [`ReplicateBase._stop_input()`](src/langchain_replicate/_base.py:168) maps to either `stop` or `stop_sequences`, and [`Replicate._stream()`](src/langchain_replicate/llms.py:145) performs local truncation/cancellation when the backend schema lacks native stop support.
- Some embedding models require non-list input shapes; use [`ReplicateEmbeddings.texts_value_mapping`](src/langchain_replicate/embeddings.py:37) (for example [`json.dumps`](json/__init__.py)) instead of reshaping request payloads inline.
- Type checking uses pyrefly strict mode, not mypy/pyright; existing ignores such as `missing-attribute`, `bad-argument-type`, and `unexpected-keyword` reflect Replicate SDK pydantic-v1 interop and dynamic kwargs patterns.
- [`src/langchain_replicate/_version.py`](src/langchain_replicate/_version.py) is generated by setuptools_scm and excluded from lint/type checks; never edit it manually.
